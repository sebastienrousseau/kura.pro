# CloudCDN — よくある質問

## はじめに

### CloudCDN とは何ですか?
CloudCDN は Git ネイティブな画像 CDN です。GitHub リポジトリに画像をプッシュするだけで、自動的に最適化(WebP + AVIF)され、Cloudflare の 300 以上のエッジロケーションから 100ms 未満のレイテンシでグローバル配信されます。

### CloudCDN は Cloudinary や Imgix と何が違うのですか?
CloudCDN は Git ネイティブのワークフローを採用しています — ダッシュボードからのアップロードも、SDK も、クレジット制度もありません。`git push` するだけでアセットが公開されます。Cloudinary は有料プランが月 89 ドルからでクレジット制課金です。Imgix は月 25 ドルからのクレジットバンドルを使用します。CloudCDN の Pro プランは月 29 ドルで、シンプルな帯域幅課金です。

### CloudCDN はオープンソースプロジェクトでも無料ですか?
はい。無料プランは月 10 GB の帯域幅で永久無料です。クレジットカード不要、試用期限もありません。OSS プロジェクトのロゴ、バナー、アイコン、ドキュメント用アセットに最適です。

### 何かインストールする必要はありますか?
いいえ。Git(2.34 以上)と署名付きコミット用の SSH キーだけがあれば十分です。Node.js もパッケージマネージャーもビルドツールも不要です。CI/CD パイプラインがすべての画像最適化をサーバー側で処理します。

## ファイル形式

### どのファイル形式に対応していますか?
アップロード可能: PNG、JPEG、SVG、ICO、WebP。パイプラインはすべての PNG および JPEG アップロードに対して WebP と AVIF のバリアントを自動生成します。SVG と ICO ファイルはそのまま配信されます。

### 自動変換ではどのような品質設定が使われますか?
WebP: 品質 80(ほぼ可逆、PNG より約 60% 軽量)。AVIF: 品質 65(高効率、PNG より約 70% 軽量)。これらは画質とファイルサイズの最適なバランスに調整されています。

### 品質設定を上書きできますか?
Pro プランでは、画像変換 API が URL パラメータでカスタム品質をサポートします: `?q=90` で高品質、`?q=50` でより圧縮します。無料プランはデフォルト設定を使用します。

### JPEG XL についてはどうですか?
2026 年現在、JPEG XL は Chrome と Firefox ではフラグの背後にあり、Safari では部分的なサポートです。ブラウザサポートが 80% を超えた時点で JPEG XL の自動変換を追加します。現在は AVIF の方が圧縮率が高く、互換性も広い(93% 以上のブラウザでサポート)です。

### 最大ファイルサイズはいくつですか?
CDN 配信では 1 ファイルあたり 25 MB です。25 MB を超えるファイルは Git リポジトリには残りますが、エッジデプロイからは除外されます。参考までに、高品質な 4K PNG は通常 5〜15 MB です。

## パフォーマンス

### アセット配信はどれくらい速いですか?
TTFB の中央値は北米とヨーロッパで 50ms 未満、グローバルで 100ms 未満です。アセットは不変キャッシュヘッダー(1 年間の max-age)で配信されるため、再訪問時はブラウザキャッシュから直接配信されます。

### キャッシュはどのように機能しますか?
すべてのアセットは `Cache-Control: public, max-age=31536000, immutable` で配信されます。ブラウザと CDN エッジは 1 年間ファイルをキャッシュします。アセットを更新するには、ファイル名を変更します(例: `logo-v2.webp`)。これがキャッシュを無効化する標準的な方法です。

### キャッシュヒット率はどれくらいですか?
本番アセットのキャッシュヒット率は通常 95% を超えます。manifest.json ファイルは新鮮さを保つために 5 分間キャッシュされます。ダッシュボードはキャッシュされません。

### キャッシュを手動でパージできますか?
無料プラン: キャッシュパージはデプロイ時に自動で行われます。Pro/Enterprise: Cloudflare ダッシュボードまたは API から特定の URL やワイルドカードパターンをパージできます。

## 画像変換 API(Pro 以上)

### 変換 API はどのように使いますか?
任意のアセット URL に URL パラメータを追加します:
```
https://cloudcdn.pro/project/image.png?w=800&h=600&fit=cover&format=auto&q=80
```

### どのようなパラメータが使えますか?
- `w` — ピクセル単位の幅(例: `?w=400`)
- `h` — ピクセル単位の高さ(例: `?h=300`)
- `fit` — リサイズモード: `cover`、`contain`、`fill`、`inside`、`outside`
- `format` — 出力形式: `auto`(ブラウザに最適)、`webp`、`avif`、`png`、`jpeg`
- `q` — 品質: 1-100(デフォルト値は形式によって異なる)
- `blur` — ガウシアンぼかし: 1-250(例: `?blur=20` で LQIP プレースホルダー)
- `sharpen` — シャープ化: 1-10
- `gravity` — トリミングのアンカー: `center`、`north`、`south`、`east`、`west`、`face`(AI)

### CloudCDN は自動形式ネゴシエーションをサポートしていますか?
はい(Pro 以上)。`?format=auto` を使用すると、CloudCDN はブラウザの `Accept` ヘッダーを読み取り、AVIF(対応している場合)、次に WebP、次にオリジナル形式の順で配信します。これにより、すべての訪問者が可能な限り小さいファイルを受け取ります。

## セットアップとワークフロー

### アセットはどのようにアップロードしますか?
```bash
git clone git@github.com:sebastienrousseau/cloudcdn.pro.git
cp my-logo.png cloudcdn.pro/my-project/images/logos/
cd cloudcdn.pro
git add my-project/
git commit -S -m "add my-project logo"
git push origin main
```
1〜2 分以内に、アセットは `https://cloudcdn.pro/my-project/images/logos/my-logo.webp` で公開されます。

### カスタムドメインを使えますか?
Pro と Enterprise プランはカスタムドメインをサポートします。カスタムドメインは Cloudflare DNS による自動 SSL プロビジョニングと CNAME 設定が行われます。設定については support@cloudcdn.pro までご連絡ください。

### macOS、Linux、WSL2 で動作しますか?
はい。Git と SSH があるすべてのプラットフォームで動作します。SSH キー生成や Git 署名設定など、プラットフォーム固有の手順はセットアップガイドを参照してください。

## セキュリティ

### 署名付きコミットは必須ですか?
はい。`main` ブランチへのすべてのプッシュには署名付きコミット(SSH Ed25519 または GPG)が必要です。これにより、すべてのアセット変更が暗号学的に検証されます。設定方法はセキュリティガイドを参照してください。

### データは安全ですか?
アセットは TLS 1.3 を使用した HTTPS で配信されます。オリジンリポジトリは GitHub にあり、ブランチ保護が設定されています。Cloudflare はすべてのプランで DDoS 保護、WAF、ボット軽減を提供します。

## 課金

### 帯域幅課金はどのように機能しますか?
無料: 10 GB/月。Pro: 100 GB/月込み、超過分は 0.05 ドル/GB。Enterprise: 無制限。帯域幅 = エッジからエンドユーザーに配信されたバイト数。オリジンプルや CI/CD 転送はカウントされません。

### いつでもアップグレード/ダウングレードできますか?
はい。アップグレードは即時有効(日割り計算)。ダウングレードは次回の請求サイクルから有効です。いつでもキャンセル可能 — 契約はありません。

### Pro の無料試用はありますか?
はい。14 日間のフルアクセス試用です。クレジットカードは不要です。サブスクライブしなければ自動的に無料プランに戻ります。

### 無料プランの帯域幅を超えるとどうなりますか?
その月の残りの期間、CDN からのアセット配信が停止されます。Git リポジトリには残り、翌月初に配信が再開されます。使用量が 80% に達した時点でメール警告が届きます。

## コンプライアンス

### CloudCDN は GDPR に準拠していますか?
はい。CloudCDN は Cloudflare のインフラを使用しており、GDPR に従ってデータを処理しています。Enterprise のお客様向けにデータ処理契約(DPA)を提供しています。個人ユーザーデータは保存されません — 静的ファイルのみを配信します。

### データはどこに保存されますか?
アセットファイルは GitHub リポジトリ(米国)に保存され、Cloudflare の 300 以上のグローバルエッジロケーションでキャッシュされます。キャッシュコピーは 1 年後または新しいデプロイ時に有効期限が切れます。
