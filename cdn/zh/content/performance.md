# CloudCDN — 性能

## 边缘网络
CloudCDN 构建在 Cloudflare 的全球网络之上：
- 跨越 100+ 个国家、193+ 个城市的 **300+ 个数据中心**。
- **全球 95% 的人口** 距离 Cloudflare PoP 不到 50ms。
- 用于最佳路由的 **8,000+ 个网络互连**。
- Anycast 路由自动将请求引导到最近的 PoP。

## 延迟
| 区域 | 中位 TTFB（缓存命中） | P95 TTFB |
|------|---------------------|----------|
| 北美 | <30ms | <80ms |
| 欧洲 | <35ms | <90ms |
| 亚太 | <50ms | <120ms |
| 南美 | <60ms | <150ms |
| 非洲 | <80ms | <200ms |

这些是边缘缓存命中时间。对 PoP 的首次请求需要源站获取（一次性增加 50-200ms）。

## 缓存策略
所有资源都使用积极的不可变缓存：
```
Cache-Control: public, max-age=31536000, immutable
```

这意味着：
- **浏览器缓存：** 1 年。无重新验证请求。
- **CDN 边缘缓存：** 1 年。从最近的 PoP 提供服务。
- **缓存命中率：** 生产资源 >95%。
- **缓存破坏：** 更改文件名/路径以提供更新的资源。

manifest.json 使用短期缓存：
```
Cache-Control: public, max-age=300
```

## 图像压缩

### 格式比较
| 格式 | 典型大小（与 PNG 相比） | 浏览器支持 | 用例 |
|------|-----------------------|-----------|------|
| PNG | 基线（100%） | 100% | 无损、透明 |
| WebP | PNG 的约 40% | 97% | 通用 Web 交付 |
| AVIF | PNG 的约 30% | 93% | 最大压缩 |
| SVG | 不一 | 100% | 矢量图形、图标 |

### 自动转换设置
- **WebP：** 质量 80，有损。质量和大小的最佳平衡。
- **AVIF：** 质量 65，有损。最大压缩，具有良好的视觉保真度。
- 原件（PNG/JPEG）始终与生成的变体一起保留。

### 真实世界的节省
对于具有 50 个图标（16x16 到 512x512）的典型项目：
- PNG 总计：约 5 MB
- WebP 总计：约 2 MB（减少 60%）
- AVIF 总计：约 1.5 MB（减少 70%）

对于横幅图像（1200x630）：
- PNG：平均约 500 KB
- WebP：平均约 150 KB
- AVIF：平均约 90 KB

## 对 Core Web Vitals 的影响
通过 CloudCDN 提供优化的图像直接改善：

### LCP（Largest Contentful Paint）
- 目标：<2.5 秒。
- 影响：对于图像密集的页面，提供 AVIF 而非 PNG 可以将 LCP 降低 40-60%。
- 提示：使用 `<link rel="preload" as="image">` 预加载首屏图像。

### CLS（Cumulative Layout Shift）
- 目标：<0.1。
- 影响：始终在 `<img>` 标签上设置 `width` 和 `height` 属性以保留空间。
- 提示：使用 manifest.json `size` 字段进行响应式布局计算。

### INP（Interaction to Next Paint）
- 目标：<200ms。
- 影响：较小的图像意味着更少的主线程解码工作。
- 提示：在首屏以下的图像上使用 `loading="lazy"` 以减少初始页面重量。

## 部署性能
| 指标 | 值 |
|------|-----|
| 增量部署（1-10 个更改文件） | 5-15 秒 |
| 完整部署（10,000+ 个文件） | 30-60 秒 |
| 图像压缩（每个 PNG，CI） | 约 200ms |
| 清单生成（10,000+ 个资源） | <5 秒 |

部署使用基于内容哈希的去重 — 仅上传新的或更改的文件。在初始部署之后，后续推送通常仅上传更改的文件。

## 监控
- **Cloudflare Analytics：** 在 Cloudflare 控制台 → Workers & Pages → cloudcdn-pro → Metrics 中可用。
- **GitHub Actions：** 构建和部署日志可在 Actions 选项卡中查看。
- **状态：** 主页（cloudcdn.pro）显示运营状态。

## 优化提示
1. 尽可能 **使用 AVIF URL** — 它们比 PNG 小 70%。
2. **使用 `<picture>` 元素** 进行格式回退（AVIF → WebP → PNG）。
3. **预加载首屏的关键图像**。
4. 使用 `loading="lazy"` **延迟加载首屏以下的所有内容**。
5. 在所有 `<img>` 标签上 **设置明确的尺寸** 以防止布局偏移。
6. **使用所需的最小图标尺寸** — 当 64x64 足够时，不要提供 512x512。
7. **Pro 套餐：** 使用 `?format=auto` 让 CloudCDN 自动提供最佳格式。
