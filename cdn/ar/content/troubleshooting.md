# CloudCDN — استكشاف الأخطاء وإصلاحها

## الأصل لا يتم تحميله (404)

### العرض
`https://cloudcdn.pro/المشروع/image.webp` يُرجع 404.

### الأسباب الشائعة
1. **الملف لم يُدفع بعد.** تحقق من `git status` — هل تم الالتزام بالملف ودفعه؟
2. **لا يزال النشر قيد التقدم.** يستغرق نشر GitHub Actions 30-90 ثانية. تحقق من علامة تبويب Actions.
3. **المسار خاطئ.** عناوين URL حساسة لحالة الأحرف. `Logo.webp` ليس `logo.webp`.
4. **لم يتم توليد WebP/AVIF بعد.** يتم تشغيل التحويل التلقائي عند الدفع. إذا دفعت ملف PNG، تظهر متغيرات `.webp` و `.avif` بعد اكتمال إجراء compress-images.
5. **الملف يزيد عن 25 ميجابايت.** يتم استبعاد الملفات التي تتجاوز 25 ميجابايت من تسليم CDN. تحقق من حجم الملف باستخدام `ls -lh`.

### الإصلاح
```bash
# تحقق من وجود الملف في المستودع
git ls-files | grep your-file

# تحقق من حالة Actions
gh run list --limit 5

# اختبر عنوان URL مباشرة
curl -sI https://cloudcdn.pro/المشروع/images/logo.webp
```

## فشل توقيع الالتزام

### العرض
```
error: Signing failed: agent refused operation
```

### الأسباب الشائعة
1. **وكيل SSH لا يعمل.** ابدأه:
   ```bash
   eval "$(ssh-agent -s)"
   ssh-add ~/.ssh/id_ed25519
   ```
2. **لم تلمس مفتاح الأجهزة.** إذا كنت تستخدم YubiKey أو مفتاح أمان (Ed25519-SK)، انقر على المفتاح عند الطلب.
3. **مفتاح توقيع خاطئ مكوّن.** تحقق:
   ```bash
   git config --global user.signingkey
   ```
4. **مفتاح SSH غير مضاف إلى GitHub.** انتقل إلى GitHub → Settings → SSH and GPG keys. تأكد من إدراج مفتاحك كـ **Signing Key** (وليس فقط Authentication).

## لم يتم توليد WebP/AVIF

### العرض
دفعت ملف PNG ولكن لم يظهر أي متغير `.webp` أو `.avif`.

### الأسباب الشائعة
1. **لم يتم تشغيل إجراء الضغط.** يتم تشغيل سير العمل فقط على ملفات PNG/JPEG الجديدة. إذا كان الملف موجودًا بالفعل، فلن تتم إعادة معالجته. تحقق من علامة تبويب Actions.
2. **لم يتم اكتشاف الملف على أنه جديد.** يستخدم سير العمل `git diff HEAD~1` للعثور على الملفات الجديدة. إذا قمت بتعديل التزام، قد لا يكتشفه diff.
3. **فشل تحويل Sharp.** يمكن أن تتسبب بعض ملفات PNG المشوهة أو ملفات تعريف الألوان غير العادية في حدوث أخطاء التحويل. تحقق من سجلات Action.

### الإصلاح
شغّل نص التحويل المحلي يدويًا:
```bash
cd scripts && npm install
node convert.mjs ../../your-project
```

## محتوى قديم بعد الدفع

### العرض
دفعت صورة محدثة ولكن لا تزال النسخة القديمة تُقدَّم.

### السبب
يتم تخزين الأصول مؤقتًا برؤوس `immutable` لمدة عام واحد. لن يؤدي تحديث ملف على نفس عنوان URL إلى إبطال ذاكرات التخزين المؤقت.

### الإصلاح
**غيّر اسم الملف أو المسار.** هذا حسب التصميم — التخزين المؤقت غير القابل للتغيير هو أسرع استراتيجية تسليم.
```bash
# بدلاً من تحديث logo.png، استخدم أسماء بإصدارات:
logo-v2.png
# أو بناءً على التاريخ:
logo-2026-03.png
```

يمكن لعملاء Pro/Enterprise مسح عناوين URL محددة عبر لوحة تحكم Cloudflare.

## فشل النشر (GitHub Actions)

### العرض
يفشل إجراء "Deploy to Cloudflare Pages".

### الأسباب الشائعة
1. **رمز API غير صالح.** قد يكون الرمز قد انتهت صلاحيته أو تم تدويره. حدّث `CLOUDFLARE_API_TOKEN` في GitHub Secrets.
2. **صلاحيات مفقودة.** يحتاج الرمز إلى: Cloudflare Pages Edit، Workers Scripts Edit، Vectorize Edit، Workers KV Storage Edit، Workers AI Read.
3. **الملف يزيد عن 25 ميجابايت.** يزيل سير عمل النشر الملفات التي تزيد عن 25 ميجابايت تلقائيًا، لكن تحقق من السجلات بحثًا عن أخطاء.
4. **مشكلة في خدمة Cloudflare.** تحقق من cloudflarestatus.com.

### الإصلاح
```bash
# أعد تشغيل سير العمل الفاشل
gh run rerun <run-id>

# تحقق من السجلات
gh run view <run-id> --log-failed
```

## البيان لا يتم تحديثه

### العرض
لا تظهر الأصول الجديدة في `manifest.json` أو لوحة التحكم.

### السبب
يتم تشغيل مولد البيان عند تغييرات مسار الصورة. إذا دفعت ملفات خارج المسارات المتوقعة، فقد لا يتم تشغيله.

### الإصلاح
شغّله يدويًا:
```bash
gh workflow run generate-manifest
```
أو قم بإعادة التوليد محليًا:
```bash
node scripts/generate-manifest.mjs
git add manifest.json
git commit -S -m "update manifest"
git push
```

## تم الوصول إلى حد النطاق الترددي (الباقة المجانية)

### العرض
تُرجع الأصول أخطاء أو تتوقف عن التحميل في منتصف الشهر.

### السبب
تحتوي الباقة المجانية على 10 جيجابايت/شهر من النطاق الترددي. ستتلقى بريدًا إلكترونيًا عند 80% من الاستخدام.

### الإصلاح
- قم بتحسين الصور بشكل أكبر (استخدم عناوين URL لـ AVIF بدلاً من PNG لتقليل ~70%).
- قم بالترقية إلى Pro (29 دولارًا/شهر) للحصول على 100 جيجابايت/شهر.
- انتظر الشهر التالي — تتم إعادة تعيين الحدود في اليوم الأول.

## محادثة Concierge لا تستجيب

### العرض
عنصر محادثة الذكاء الاصطناعي على الصفحة الرئيسية لا يستجيب أو يعرض أخطاء.

### الأسباب الشائعة
1. **تم الوصول إلى حد الاستعلامات الشهرية (1,000/شهر).** يقوم Concierge بتعطيل نفسه عند الوصول إلى الحد.
2. **Cloudflare Workers AI غير متاح مؤقتًا.** نادر، لكن استدلال الذكاء الاصطناعي للحافة يمكن أن يشهد انقطاعات قصيرة.
3. **قاعدة المعرفة غير متزامنة.** إذا تم تحديث ملفات المحتوى مؤخرًا، فقد يحتاج فهرس Vectorize إلى إعادة المزامنة.

### الإصلاح
لمشاكل مزامنة المعرفة:
```bash
CLOUDFLARE_API_TOKEN=<token> CLOUDFLARE_ACCOUNT_ID=<id> node scripts/sync-knowledge.mjs cdn/content
```
