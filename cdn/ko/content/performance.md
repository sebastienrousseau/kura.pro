# CloudCDN — 성능

## 엣지 네트워크
CloudCDN은 Cloudflare의 글로벌 네트워크 위에 구축되었습니다:
- 100개 이상의 국가, 193개 이상의 도시에 걸친 **300개 이상의 데이터 센터**.
- **전 세계 인구의 95%**가 Cloudflare PoP에서 50ms 이내에 있습니다.
- 최적의 라우팅을 위한 **8,000개 이상의 네트워크 상호 연결**.
- 애니캐스트 라우팅이 가장 가까운 PoP로 요청을 자동으로 안내합니다.

## 지연 시간
| 지역 | 중앙값 TTFB (캐시 히트) | P95 TTFB |
|------|------------------------|----------|
| 북미 | <30ms | <80ms |
| 유럽 | <35ms | <90ms |
| 아시아 태평양 | <50ms | <120ms |
| 남미 | <60ms | <150ms |
| 아프리카 | <80ms | <200ms |

이는 엣지 캐시 히트 시간입니다. PoP에 대한 첫 번째 요청은 원본 가져오기가 필요합니다(한 번에 50-200ms 추가).

## 캐싱 전략
모든 에셋은 적극적인 불변 캐싱을 사용합니다:
```
Cache-Control: public, max-age=31536000, immutable
```

이는 다음을 의미합니다:
- **브라우저 캐시:** 1년. 재검증 요청 없음.
- **CDN 엣지 캐시:** 1년. 가장 가까운 PoP에서 제공됨.
- **캐시 적중률:** 프로덕션 에셋의 경우 95% 이상.
- **캐시 무효화:** 업데이트된 에셋을 제공하려면 파일명/경로를 변경하십시오.

manifest.json은 단기 캐싱을 사용합니다:
```
Cache-Control: public, max-age=300
```

## 이미지 압축

### 형식 비교
| 형식 | 일반 크기 (PNG 대비) | 브라우저 지원 | 사용 사례 |
|------|---------------------|--------------|----------|
| PNG | 기준 (100%) | 100% | 무손실, 투명도 |
| WebP | PNG의 ~40% | 97% | 일반 웹 전송 |
| AVIF | PNG의 ~30% | 93% | 최대 압축 |
| SVG | 다양함 | 100% | 벡터 그래픽, 아이콘 |

### 자동 변환 설정
- **WebP:** 품질 80, 손실 압축. 품질과 크기의 최적 균형.
- **AVIF:** 품질 65, 손실 압축. 좋은 시각적 충실도와 함께 최대 압축.
- 원본(PNG/JPEG)은 항상 생성된 변형과 함께 보존됩니다.

### 실제 절약
50개의 아이콘(16x16부터 512x512)이 있는 일반 프로젝트의 경우:
- PNG 총합: ~5 MB
- WebP 총합: ~2 MB (60% 감소)
- AVIF 총합: ~1.5 MB (70% 감소)

배너 이미지(1200x630)의 경우:
- PNG: 평균 ~500 KB
- WebP: 평균 ~150 KB
- AVIF: 평균 ~90 KB

## Core Web Vitals 영향
CloudCDN을 통해 최적화된 이미지를 제공하면 다음이 직접적으로 개선됩니다:

### LCP (Largest Contentful Paint)
- 목표: <2.5초.
- 영향: PNG 대신 AVIF를 제공하면 이미지가 많은 페이지의 LCP를 40-60% 줄일 수 있습니다.
- 팁: `<link rel="preload" as="image">`로 above-the-fold 이미지를 미리 로드하십시오.

### CLS (Cumulative Layout Shift)
- 목표: <0.1.
- 영향: 공간을 예약하기 위해 항상 `<img>` 태그에 `width` 및 `height` 속성을 설정하십시오.
- 팁: 반응형 레이아웃 계산에 manifest.json `size` 필드를 사용하십시오.

### INP (Interaction to Next Paint)
- 목표: <200ms.
- 영향: 더 작은 이미지는 메인 스레드 디코딩 작업이 더 적음을 의미합니다.
- 팁: 초기 페이지 무게를 줄이기 위해 below-the-fold 이미지에 `loading="lazy"`를 사용하십시오.

## 배포 성능
| 지표 | 값 |
|------|-----|
| 증분 배포 (1-10개 변경 파일) | 5-15초 |
| 전체 배포 (10,000개 이상 파일) | 30-60초 |
| 이미지 압축 (PNG당, CI) | ~200ms |
| 매니페스트 생성 (10,000개 이상 에셋) | <5초 |

배포는 콘텐츠 해시 중복 제거를 사용합니다 — 새로운 또는 변경된 파일만 업로드됩니다. 초기 배포 후 후속 푸시는 일반적으로 변경된 파일만 업로드합니다.

## 모니터링
- **Cloudflare Analytics:** Cloudflare 대시보드 → Workers & Pages → cloudcdn-pro → Metrics에서 사용 가능.
- **GitHub Actions:** Actions 탭에서 빌드 및 배포 로그를 사용할 수 있습니다.
- **상태:** 홈페이지(cloudcdn.pro)에 운영 상태가 표시됩니다.

## 최적화 팁
1. 가능한 경우 **AVIF URL 사용** — PNG보다 70% 더 작습니다.
2. 형식 폴백을 위해 **`<picture>` 요소 사용** (AVIF → WebP → PNG).
3. above-the-fold의 **중요한 이미지 미리 로드**.
4. `loading="lazy"`로 **below-the-fold의 모든 것을 지연 로드**.
5. 레이아웃 시프트를 방지하기 위해 모든 `<img>` 태그에 **명시적 치수 설정**.
6. **필요한 가장 작은 아이콘 크기 사용** — 64x64로 충분할 때 512x512를 제공하지 마십시오.
7. **Pro 등급:** CloudCDN이 자동으로 최적 형식을 제공하도록 `?format=auto`를 사용하십시오.
