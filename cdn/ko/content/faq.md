# CloudCDN — 자주 묻는 질문

## 시작하기

### CloudCDN이란 무엇입니까?
CloudCDN은 Git 네이티브 이미지 CDN입니다. GitHub 리포지토리에 이미지를 푸시하면 자동으로 최적화(WebP + AVIF)되어 Cloudflare의 300개 이상 엣지 위치에서 100ms 미만의 대기 시간으로 전 세계에 제공됩니다.

### CloudCDN은 Cloudinary나 Imgix와 어떻게 다릅니까?
CloudCDN은 Git 네이티브 워크플로를 사용합니다 — 대시보드 업로드 없음, SDK 없음, 크레딧 시스템 없음. `git push`만 하면 에셋이 라이브로 제공됩니다. Cloudinary는 유료 플랜의 경우 월 $89부터 시작하는 크레딧 기반 청구를 사용합니다. Imgix는 월 $25부터 시작하는 크레딧 번들을 사용합니다. CloudCDN의 Pro 등급은 월 $29이며 단순한 대역폭 청구를 사용합니다.

### CloudCDN은 오픈소스 프로젝트에 무료입니까?
예. 무료 등급은 월 10 GB 대역폭으로 영구적으로 무료입니다. 신용 카드 불필요, 평가판 만료 없음. OSS 프로젝트 로고, 배너, 아이콘 및 문서 에셋에 이상적입니다.

### 무엇인가 설치해야 합니까?
아니요. Git(2.34+)과 서명된 커밋을 위한 SSH 키만 필요합니다. Node.js 없음, 패키지 매니저 없음, 빌드 도구 없음. CI/CD 파이프라인이 모든 이미지 최적화를 서버 측에서 처리합니다.

## 파일 형식

### 어떤 파일 형식이 지원됩니까?
업로드: PNG, JPEG, SVG, ICO, WebP. 파이프라인은 모든 PNG 및 JPEG 업로드에 대해 WebP 및 AVIF 변형을 자동으로 생성합니다. SVG 및 ICO 파일은 그대로 제공됩니다.

### 자동 변환에는 어떤 품질 설정이 사용됩니까?
WebP: 품질 80(거의 무손실, PNG보다 ~60% 작음). AVIF: 품질 65(높은 효율, PNG보다 ~70% 작음). 이는 시각적 품질과 파일 크기의 최적 균형을 위해 최적화되었습니다.

### 품질 설정을 재정의할 수 있습니까?
Pro 등급에서 이미지 변환 API는 URL 매개변수를 통한 사용자 정의 품질을 지원합니다: 더 높은 품질을 위해 `?q=90`, 더 많은 압축을 위해 `?q=50`. 무료 등급은 기본 설정을 사용합니다.

### JPEG XL은 어떻습니까?
2026년 현재 JPEG XL은 Chrome과 Firefox에서 플래그 뒤에 있으며 Safari에서 부분 지원됩니다. 브라우저 지원이 80% 이상에 도달하면 JPEG XL 자동 변환을 추가할 것입니다. 현재 AVIF는 더 넓은 호환성(브라우저 지원 93% 이상)으로 더 나은 압축을 제공합니다.

### 최대 파일 크기는 얼마입니까?
CDN 전송용 파일당 25 MB. 25 MB 이상의 파일은 Git 리포지토리에 남아 있지만 엣지 배포에서 제외됩니다. 참고로 고품질 4K PNG는 일반적으로 5-15 MB입니다.

## 성능

### 에셋 전송 속도는 얼마나 빠릅니까?
북미 및 유럽에서는 중앙값 TTFB 50ms 미만, 전 세계적으로 100ms 미만. 에셋은 불변 캐시 헤더(1년 max-age)와 함께 제공되므로 반복 방문은 브라우저 캐시에서 직접 제공됩니다.

### 캐시는 어떻게 작동합니까?
모든 에셋은 `Cache-Control: public, max-age=31536000, immutable`과 함께 제공됩니다. 브라우저와 CDN 엣지는 파일을 1년 동안 캐시합니다. 에셋을 업데이트하려면 파일 이름을 변경하십시오(예: `logo-v2.webp`) — 이것이 표준 캐시 무효화 접근 방식입니다.

### 캐시 적중률은 얼마입니까?
프로덕션 에셋은 일반적으로 95% 이상의 캐시 적중률을 보입니다. manifest.json 파일은 신선함을 유지하기 위해 5분 동안 캐시됩니다. 대시보드는 절대 캐시되지 않습니다.

### 캐시를 수동으로 퍼지할 수 있습니까?
무료 등급: 캐시 퍼지는 배포 시 자동으로 발생합니다. Pro/Enterprise: Cloudflare 대시보드 또는 API를 통해 특정 URL 또는 와일드카드 패턴을 퍼지할 수 있습니다.

## 이미지 변환 API (Pro+)

### 변환 API는 어떻게 작동합니까?
모든 에셋 URL에 URL 매개변수를 추가하십시오:
```
https://cloudcdn.pro/프로젝트/image.png?w=800&h=600&fit=cover&format=auto&q=80
```

### 어떤 매개변수를 사용할 수 있습니까?
- `w` — 픽셀 단위 너비 (예: `?w=400`)
- `h` — 픽셀 단위 높이 (예: `?h=300`)
- `fit` — 크기 조정 모드: `cover`, `contain`, `fill`, `inside`, `outside`
- `format` — 출력 형식: `auto`(브라우저에 최적), `webp`, `avif`, `png`, `jpeg`
- `q` — 품질: 1-100(기본값은 형식에 따라 다름)
- `blur` — 가우시안 블러: 1-250 (예: LQIP 자리표시자용 `?blur=20`)
- `sharpen` — 선명화: 1-10
- `gravity` — 자르기 기준점: `center`, `north`, `south`, `east`, `west`, `face`(AI)

### CloudCDN은 자동 형식 협상을 지원합니까?
예 (Pro+). `?format=auto`를 사용하면 CloudCDN은 브라우저의 `Accept` 헤더를 읽고 AVIF(지원되는 경우), 그 다음 WebP, 그 다음 원본 형식을 제공합니다. 이것은 모든 방문자가 가능한 가장 작은 파일을 받도록 보장합니다.

## 설정 및 워크플로

### 에셋을 어떻게 업로드합니까?
```bash
git clone git@github.com:sebastienrousseau/cloudcdn.pro.git
cp my-logo.png cloudcdn.pro/my-project/images/logos/
cd cloudcdn.pro
git add my-project/
git commit -S -m "add my-project logo"
git push origin main
```
1-2분 이내에 `https://cloudcdn.pro/my-project/images/logos/my-logo.webp`에서 에셋이 라이브가 됩니다.

### 사용자 정의 도메인을 사용할 수 있습니까?
Pro 및 Enterprise 등급은 사용자 정의 도메인을 지원합니다. 사용자 정의 도메인은 Cloudflare DNS를 통한 자동 SSL 프로비저닝과 CNAME 설정을 받습니다. 구성하려면 support@cloudcdn.pro로 연락하십시오.

### macOS, Linux 및 WSL2에서 작동합니까?
예. Git과 SSH가 있는 모든 플랫폼. SSH 키 생성 및 Git 서명 구성을 포함한 플랫폼별 지침은 설정 가이드를 참조하십시오.

## 보안

### 서명된 커밋이 필수입니까?
예. main 브랜치로의 모든 푸시는 서명된 커밋(SSH Ed25519 또는 GPG)이 필요합니다. 이는 모든 에셋 변경이 암호학적으로 검증되도록 보장합니다. 설정은 보안 가이드를 참조하십시오.

### 제 데이터는 안전합니까?
에셋은 TLS 1.3을 사용하는 HTTPS를 통해 제공됩니다. 원본 리포지토리는 브랜치 보호가 적용된 GitHub에 있습니다. Cloudflare는 모든 플랜에서 DDoS 보호, WAF 및 봇 완화를 제공합니다.

## 청구

### 대역폭 청구는 어떻게 작동합니까?
무료: 월 10 GB. Pro: 월 100 GB 포함, GB당 $0.05 초과 요금. Enterprise: 무제한. 대역폭 = 엣지에서 최종 사용자에게 전달된 총 바이트. 원본 풀과 CI/CD 전송은 계산되지 않습니다.

### 언제든지 업그레이드하거나 다운그레이드할 수 있습니까?
예. 업그레이드는 즉시 적용됩니다(일할 계산). 다운그레이드는 다음 청구 주기에 적용됩니다. 언제든지 취소 — 계약 없음.

### Pro 무료 평가판이 있습니까?
예. 14일 전체 액세스 평가판. 신용 카드 불필요. 구독하지 않으면 자동으로 무료 등급으로 되돌아갑니다.

### 무료 등급 대역폭을 초과하면 어떻게 됩니까?
에셋은 해당 월의 나머지 기간 동안 CDN에서 제공이 중단됩니다. Git 리포지토리에 남아 있다가 다음 달이 시작될 때 다시 제공됩니다. 80% 사용 시 이메일 경고를 받게 됩니다.

## 컴플라이언스

### CloudCDN은 GDPR을 준수합니까?
예. CloudCDN은 GDPR에 따라 데이터를 처리하는 Cloudflare의 인프라를 사용합니다. Enterprise 고객을 위한 데이터 처리 계약(DPA)을 제공합니다. 개인 사용자 데이터는 저장되지 않습니다 — 정적 파일만 제공합니다.

### 데이터는 어디에 저장됩니까?
에셋 파일은 GitHub 리포지토리(미국)에 저장되고 Cloudflare의 300개 이상 글로벌 엣지 위치에 캐시됩니다. 캐시된 사본은 1년 후 또는 새로운 배포 시 만료됩니다.
