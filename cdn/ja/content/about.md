# CloudCDN について

## CloudCDN とは
CloudCDN は開発者向けの Git ネイティブな静的アセット配信ネットワークです。GitHub リポジトリに画像をプッシュするだけで自動的に最適化され、Cloudflare のエッジネットワーク(300 以上のデータセンター、100 以上の国、100ms 未満のレイテンシ)から世界中に配信されます。

ダッシュボードやアップロード API、SDK 統合を必要とする従来の画像 CDN とは異なり、CloudCDN は開発者が既に慣れ親しんだワークフロー、すなわち `git push` をそのまま活用します。

## 仕組み
1. **プッシュ:** GitHub リポジトリに画像を追加し、署名付きキーでコミットします。
2. **最適化:** GitHub Actions が画像を自動的に圧縮し、WebP(品質 80、約 60% 軽量化)および AVIF(品質 65、約 70% 軽量化)のバリアントを生成します。
3. **デプロイ:** 変更されたファイルが増分デプロイで Cloudflare Pages にアップロードされます(コンテンツハッシュによる重複排除 ― 新規または変更されたファイルのみが転送されます)。
4. **配信:** アセットは不変キャッシュヘッダー(`max-age=31536000`)とともに、300 以上のエッジロケーションの中で最も近い場所から配信されます。

## テクノロジースタック
- **エッジネットワーク:** Cloudflare Pages による世界 300 以上の PoP。世界人口の 95% が Cloudflare データセンターから 50ms 圏内に位置しています。
- **画像形式:** PNG(可逆圧縮オリジナル)、WebP(非可逆、品質 80)、AVIF(非可逆、品質 65)、SVG(そのまま配信)、ICO(そのまま配信)。
- **フォーマットネゴシエーション:** Pro プランではブラウザの `Accept` ヘッダーに基づいて最適な形式を配信します(AVIF > WebP > オリジナル)。
- **キャッシュ:** エッジおよびブラウザでの不変キャッシュ、max-age は 1 年。キャッシュ無効化はファイル名またはパスの変更で行います。
- **CI/CD:** 圧縮(Sharp)、マニフェスト生成、Cloudflare Pages デプロイ(Wrangler)に GitHub Actions を使用。
- **セキュリティ:** SSH Ed25519 による署名付きコミットを必須化。`main` ブランチの保護設定。API トークンは GitHub Secrets で暗号化管理。
- **AI コンシェルジュ:** Cloudflare Workers AI(Llama 3.1)と Vectorize RAG によるトップページ上のインテリジェントなドキュメント検索。

## パフォーマンス
- **TTFB:** 北米およびヨーロッパで中央値 50ms 未満、グローバルで 100ms 未満(Cloudflare エッジキャッシュヒット時)。
- **キャッシュヒット率:** 本番アセットで 95% 以上(不変キャッシュ)。
- **デプロイ速度:** 増分アップロード ― 変更されたファイルのみを転送。通常のデプロイは 5 ~ 30 秒。
- **圧縮:** WebP は PNG より約 60% 軽量化。AVIF は PNG より約 70% 軽量化。どちらも自動生成されます。

## アセットの構成
```
project-name/
  images/
    banners/    — ワイドフォーマット画像(1200x630 推奨)
    icons/      — 複数解像度(16x16 から 512x512、@2x/@3x 対応)
    logos/      — ブランドロゴとマーク
    github/     — ソーシャルプレビュー用画像
    titles/     — タイトル画像およびヘッダー
  README.md     — プロジェクトの説明(任意)
```

## 主な指標
- **1,400 以上の最適化済みアセット**を 54 のテナントゾーンに配置。
- **画像 1 枚につき単一のソース** ― バリアントは `/api/transform` でオンデマンド生成。
- **300 以上のエッジ PoP** が 100 以上の国に配置。
- エッジキャッシュヒット時に**グローバル TTFB が 100ms 未満**。
- **ビルドステップ不要** ― `npm install` も Webpack もフレームワークも不要。

## CloudCDN を使っている人たち
CloudCDN は以下のアセット配信に利用されています:
- **オープンソースプロジェクト:** ロゴ、バナー、アイコン、ドキュメント用グラフィック。
- **開発者ツール:** Rust、Python、AI 開発者ブランディング(rustdev、pythondev、llamadev)。
- **フィンテックプラットフォーム:** 銀行および量子コンピューティングのアセットライブラリ。
- **オーディオアプリケーション:** 波形表示と UI コンポーネント。
- **静的サイトジェネレーター:** Shokunin、Kaishi などの SSG フレームワーク。

## なぜ [競合] ではないのか

| vs. Cloudinary | vs. Imgix | vs. Bunny CDN |
|---|---|---|
| 複雑なクレジット制度なし | クレジット課金なし | AVIF サポートを標準搭載 |
| Git ネイティブなワークフロー | Git ネイティブなワークフロー | Git ネイティブなワークフロー |
| 無料枠あり、クレジットカード不要 | 無料枠あり | トライアル期限なし |
| AI コンシェルジュを内蔵 | 標準ドキュメントのみ | 標準ドキュメントのみ |
